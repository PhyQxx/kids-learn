## 一、产品概述

### 1.1 产品背景

随着智能设备的普及，儿童接触移动应用的时间逐年提前。3-12岁儿童正处于认知发展的关键时期，通过游戏化方式学习认字、拼音、数学、英语等知识，已被证明能够显著提升学习兴趣和效果。目前市场上针对这一年龄段的学习应用虽多，但普遍存在以下问题：

- 游戏与学习割裂，孩子只玩不学
- 内容同质化严重，缺乏系统性
- 适龄性不足，低龄段产品过于成人化
- 家长管控功能薄弱

### 1.2 产品定位

**产品名称**：`趣学星球`（KidsLearn）

**一句话描述**：专为3-12岁儿童打造的游戏化学习平台，让孩子在趣味闯关中掌握知识。

**核心定位**：

- **学习场景**：幼儿园+小学阶段的全科学习（语文/数学/英语/逻辑/科学）
- **游戏化程度**：深度融合，而非"游戏+学习"两层皮
- **平台重心**：移动端优先（手机+Pad），适配竖屏/横屏

### 1.3 目标用户

| 维度 | 描述 |
|------|------|
| 主要用户 | 3-12岁儿童（幼儿园小班~小学六年级） |
| 家长用户 | 25-40岁家长，负责账号管理、学习监督、时间控制 |
| 使用场景 | 亲子共学、孩子独立学习、课堂辅助 |
| 用户分层 | 3-6岁（幼幼组）、6-9岁（低龄组）、9-12岁（高年龄组） |

### 1.4 核心价值

| 核心价值 | 说明 |
|----------|------|
| **适龄匹配** | 按年龄自动推荐内容，3岁和12岁看到完全不同 |
| **游戏即学习** | 每个知识点都是一个游戏关卡，闯关=学习 |
| **正向激励** | 贴纸、徽章、宠物养成让孩子获得持续成就感 |
| **家长放心** | 严格的内容审核、时间管理、使用报告 |

## 二、产品架构设计

### 2.1 技术选型

| 层级 | 技术方案                     | 说明 |
|------|--------------------------|------|
| 移动端框架 | UniApp + Vue3            | 跨平台（iOS/Android/H5/小程序），一套代码多端运行 |
| 前端UI | TuniaoUI                 | 符合儿童友好设计的组件库 |
| 管理后台前端 | Vue3 + Element Plus      | 后台管理系统界面 |
| 后端框架 | Spring Boot 3.x + JDK 17 | 现代化Java后端 |
| 数据库 | MySQL 8.x                | 主数据存储 |
| 缓存 | Redis                    | 会话缓存、排行榜、限流 |
| 文件存储 | 本地/FTP                   | 教材音频/图片/视频存储 |
| 实时通信 | WebSocket                | 实时对战、直播课 |
| AI能力 | 语音评测、图像识别                | 口语打分、作文批改（对接第三方） |
| **离线支持** | **IndexedDB（本地缓存）**      | **课程/题目/宠物数据本地存储，支持无网学习** |
| **内容对接** | **HTTP API / 文件导入**      | **支持第三方教育内容导入（火花学院等）** |

## 三、功能模块设计

### 3.1 年龄段分层与内容匹配

#### 3.1.1 幼幼组：3-6岁（幼儿园小班~大班）

**认知特点**：

- 以具象思维为主，无法理解抽象概念
- 注意力集中时间短（5-15分钟）
- 喜欢重复、简单、有反馈的操作
- 依赖家长陪伴

**推荐内容类型**：

| 内容类型 | 具体形式 | 单次时长 |
|----------|----------|----------|
| 认字 | 图文配对、拖拽拼字、点读发声 | 3-5分钟 |
| 拼音 | 声母韵母点读、跟读 | 3-5分钟 |
| 数学启蒙 | 数数、形状识别、简单大小比较 | 3-5分钟 |
| 英语启蒙 | 单词图片卡、跟读、律动儿歌 | 3-5分钟 |
| 逻辑训练 | 颜色分类、简单排序、找规律 | 3-5分钟 |
| 音乐律动 | 儿歌跟唱、节奏游戏 | 3-5分钟 |

**游戏化设计**：

- 宠物养成（喂食、换装）
- 贴纸收集（每关获得新贴纸）
- 简单闯关（3颗星评价）
- 音效+动画+震动反馈

#### 3.1.2 低年龄组：6-9岁（小学1-3年级）

**认知特点**：

- 开始具备逻辑思维能力
- 竞争意识增强，喜欢排行榜
- 自主学习能力逐步形成
- 对"奖励"敏感

**推荐内容类型**：

| 内容类型 | 具体形式 | 单次时长 |
|----------|----------|----------|
| 语文 | 识字、拼音、组词、成语故事 | 10-15分钟 |
| 数学 | 计算训练、应用题、简单几何 | 10-15分钟 |
| 英语 | 单词学习、跟读练习、情景对话 | 10-15分钟 |
| 逻辑 | 数独入门、简单编程拼搭 | 10-15分钟 |
| 科学 | 自然百科、简单实验 | 10-15分钟 |

**游戏化设计**：

- 宠物升级进化
- 成就徽章收集
- 班级排行榜
- 挑战赛（好友对战）
- 虚拟货币+商店

#### 3.1.3 高年龄组：9-12岁（小学4-6年级）

**认知特点**：

- 抽象思维能力成熟
- 自主意识强，有社交需求
- 愿意接受挑战
- 对"公平性"敏感

**推荐内容类型**：

| 内容类型 | 具体形式 | 单次时长 |
|----------|----------|----------|
| 语文 | 诗词背诵、阅读理解、写作训练 | 15-20分钟 |
| 数学 | 计算速度训练、思维题、专项突破 | 15-20分钟 |
| 英语 | 单词记忆、语法基础、阅读理解 | 15-20分钟 |
| 科学 | 初中预科、自然探索 | 15-20分钟 |
| 编程 | 图形化编程、Python入门 | 15-20分钟 |

**游戏化设计**：

- 成就称号体系
- 段位排名系统
- 公会/战队
- 赛季制挑战

#### 3.2.2 学习模块

**功能描述**：各学科的学习内容，按关卡递进。

```
选择年级 → 选择单元 → 选择知识点 → 关卡列表 → 开始学习 → 闯关 → 结果页
                                                          ↓
                                                    ⭐⭐⭐评价
                                                    解锁下一关
```

**关卡结构**（以"认识数字"为例）：

| 步骤 | 内容 | 时长 |

| 3 | 跟读练习（跟读"一"） | 30秒 |
| 4 | 互动游戏（点选正确的数量） | 60秒 |
| 5 | 总结巩固（快速回顾） | 30秒 |

**宠物互动**：

- 喂食（消耗金币）
- 换装（解锁外观）
- 玩耍（迷你游戏）
- 合影（生成分享图）

## 四、游戏化机制详细设计

### 4.1 奖励体系

#### 4.1.1 虚拟货币

**金币**：

- 获取途径：闯关、每日任务、成就奖励、挑战赛

- 获取途径：充值、年度成就、特殊活动
- 用途：购买稀有宠物、限定装饰

| 传说 | 10% | 收集册+称号 |

#### 4.1.3 奖励规则

| 行为 | 金币 | 经验 | 贴纸 | 备注 |
|------|------|------|------|------|

## 八、后台管理系统设计

### 8.1 概述

后台管理系统（kidslearn-admin）是面向运营人员和管理员的内容运营平台，核心目标是**让非技术人员可以自主维护和扩展平台内容**，无需发版即可完成学习资料、宠物系统、成就系统的配置。

### 8.2 管理后台功能架构

```
┌──────────────────────────────────────────────────────────────┐
│                     后台管理系统（Admin）                      │
├──────────┬──────────┬───────────┬───────────┬────────────────┤
│ 内容管理  │ 宠物管理  │ 成就管理   │ 运营管理   │  系统管理      │
├──────────┼──────────┼───────────┼───────────┼────────────────┤
│ 学科管理  │ 宠物种类  │ 成就定义   │ 活动管理   │  用户管理      │
│ 年级管理  │ 进化配置  │ 条件配置   │ Banner    │  角色权限      │
│ 课程管理  │ 道具管理  │ 奖励配置   │ 公告管理   │  数据统计      │
│ 关卡管理  │ 服装/装饰 │ 称号管理   │ 付费配置   │  系统配置      │
│ 题目管理  │ 外观资源  │ 贴纸管理   │ 内容审核   │  操作日志      │
└──────────┴──────────┴───────────┴───────────┴────────────────┘
```

### 8.3 内容管理模块

#### 8.3.1 学科管理

管理员可自定义学科分类，支持新增/编辑/启用/禁用。

| 操作 | 说明 |
|------|------|
| 新增学科 | 填写学科名称、代码、图标、主题色、排序权重 |
| 编辑学科 | 修改学科信息和展示样式 |
| 启用/禁用 | 禁用后客户端不再展示该学科（已有学习记录保留） |
| 排序调整 | 拖拽调整学科展示顺序 |

**管理界面字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| 学科代码 | 输入框 | 唯一标识，如 CHINESE、MATH |
| 学科名称 | 输入框 | 展示名称，如"语文" |
| 图标 | 图片上传 | 学科图标（建议 128x128 PNG） |
| 主题色 | 颜色选择器 | 学科主题色值 |
| 排序权重 | 数字输入 | 越小越靠前 |
| 状态 | 开关 | 启用/禁用 |

#### 8.3.2 课程管理

支持课程的完整生命周期管理。

**课程编辑页面**：

| 字段 | 类型 | 说明 |
|------|------|------|
| 课程名称 | 输入框 | 如"认识数字" |
| 所属学科 | 下拉选择 | 关联学科表 |
| 适用年级 | 多选 | 可同时适配多个年级 |
| 课程描述 | 富文本 | 课程介绍，支持图文 |
| 封面图 | 图片上传 | 建议尺寸 750x420 |
| 难度 | 单选 | 简单/普通/困难 |
| 是否精英关 | 开关 | 精英关卡需付费或特殊条件解锁 |
| 排序权重 | 数字输入 | 同年级内的排序 |

**课程列表支持**：
- 按学科、年级、状态筛选
- 批量启用/禁用
- 快速复制课程（作为模板创建新课程）
- 课程预览（模拟客户端展示效果）

#### 8.3.3 关卡管理

每个课程下可创建多个关卡，关卡支持排序和条件解锁。

| 字段 | 类型 | 说明 |
|------|------|------|
| 关卡序号 | 自动/手动 | 关卡在课程中的顺序 |
| 关卡名称 | 输入框 | 如"第1关：认识数字1" |
| 关卡描述 | 富文本 | 关卡说明 |
| 封面图 | 图片上传 | 关卡封面 |
| 题目数量 | 只读 | 显示已关联的题目数 |
| 及格分数 | 数字输入 | 默认60 |
| 星级门槛 | 三个数字 | 如"60,80,100"分别对应1/2/3星 |
| 经验奖励 | 数字输入 | 通关获得经验 |
| 金币奖励 | 数字输入 | 通关获得金币 |
| 通关贴纸 | 下拉选择 | 通关奖励的贴纸，可选"无" |
| 解锁条件 | JSON编辑器 | 如 `{"preLevelId":101,"minStars":2}` |

**关卡内容编排**（可视化拖拽排序）：

```
┌─────────────────────────────────────┐
│  第1关：认识数字1                    │
│  ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐    │
│  │ 1 │ │ 2 │ │ 3 │ │ 4 │ │ 5 │    │
│  │动画│ │配对│ │跟读│ │互动│ │总结│    │
│  └───┘ └───┘ └───┘ └───┘ └───┘    │
│     ← 拖拽调整题目顺序 →            │
│  [+ 添加题目]  [批量导入]           │
└─────────────────────────────────────┘
```

#### 8.3.4 题目管理

题目是内容管理的核心，支持多种题型的创建和编辑。

**支持的题型**：

| 题型 | 说明 | 编辑方式 |
|------|------|----------|
| 选择题 | 单选/多选，带选项 | 可视化选项编辑器 |
| 判断题 | 对/错 | 简洁编辑器 |
| 填空题 | 输入答案 | 设置正确答案+同义词 |
| 连线题 | 左右配对 | 拖拽连线编辑器 |
| 拖拽题 | 拖拽元素到目标位置 | 可视化拖拽编辑器 |
| 点读题 | 点击图片特定区域 | 上传图片+标注区域 |

**题目编辑器**：

| 字段 | 类型 | 说明 |
|------|------|------|
| 题型 | 下拉选择 | 选择/判断/填空/连线/拖拽 |
| 题目内容 | 富文本 | 支持文本+图片+音频 |
| 选项（选择题） | 动态列表 | A/B/C/D选项，可增减 |
| 正确答案 | 根据题型 | 选项/文本/区域坐标 |
| 解析 | 富文本 | 答题后展示的解析内容 |
| 难度 | 单选 | 简单/普通/困难 |
| 分值 | 数字输入 | 默认10分 |
| 限时 | 数字输入 | 0表示不限时 |
| 排序 | 数字输入 | 同关卡内的题目顺序 |

**批量操作**：
- **Excel导入**：下载模板 → 填写题目 → 上传导入
- **JSON导入**：支持标准JSON格式的题目批量导入
- **批量修改**：选中多个题目批量修改难度/分值/状态
- **题库复用**：从其他关卡/课程引用题目

**Excel导入模板**：

| 列 | 说明 | 示例 |
|------|------|------|
| 题型 | 选择/判断/填空 | 选择 |
| 题目内容 | 纯文本 | "这是几个苹果？🍎🍎🍎" |
| 选项A | 选项内容 | "1个" |
| 选项B | 选项内容 | "2个" |
| 选项C | 选项内容 | "3个" |
| 选项D | 选项内容 | "" |
| 正确答案 | 正确选项 | C |
| 解析 | 答案解析 | "🍎🍎🍎表示3个苹果" |
| 难度 | 简单/普通/困难 | 简单 |
| 分值 | 数字 | 10 |

#### 8.3.5 内容审核流程

```
创建/编辑 → 保存草稿 → 提交审核 → 审核通过 → 发布上线
                         ↓
                      审核驳回 → 修改 → 重新提交
```

| 状态 | 说明 |
|------|------|
| 草稿 | 编辑中，未提交 |
| 待审核 | 已提交，等待审核 |
| 已通过 | 审核通过，等待发布 |
| 已发布 | 已上线，客户端可见 |
| 已驳回 | 审核不通过，需修改 |
| 已下线 | 已发布后手动下线 |

成就的触发条件通过条件引擎配置，支持多条件组合。

**条件类型**：

| 条件代码 | 说明 | 参数示例 |
|----------|------|----------|
| DAILY_LOGIN | 每日登录 | 连续天数：7 |
| COMPLETE_LEVEL | 完成关卡 | 关卡数：10 |
| GET_STAR | 获得星星 | 总星数：50 |
| SUBJECT_COMPLETE | 完成学科 | 学科ID + 关卡数 |
| PET_LEVEL_UP | 宠物升级 | 等级：5 |
| FRIEND_COUNT | 好友数量 | 数量：10 |
| CHALLENGE_WIN | 挑战赛获胜 | 次数：5 |
| STICKER_COLLECT | 收集贴纸 | 数量：20 |
| LEARN_TIME | 学习时长 | 累计分钟：300 |
| CORRECT_RATE | 正确率 | 百分比：90 |

**条件组合规则**（JSON配置）：

```json
{
  "logic": "AND",
  "conditions": [
    {"type": "COMPLETE_LEVEL", "value": 10},
    {"type": "GET_STAR", "value": 30}
  ]
}
```

支持 `AND`（全部满足）和 `OR`（任一满足）逻辑组合。

## 九、项目规划

### 9.1 实现策略

**核心原则：逐个实现每个功能模块，而非按阶段交付应用层**

每个模块的开发顺序：

| 序号 | 模块 | 包含内容 | 优先级 |
|------|------|----------|--------|
| 1 | 用户系统 | 注册/登录/家庭组/个人档案 | P0 |
| 2 | 学习引擎 | 课程/关卡/题目/答题/记录 | P0 |
| 3 | 宠物养成 | 宠物状态/喂食/进化/背包 | P1 |
| 4 | 成就系统 | 成就定义/进度/奖励 | P1 |
| 5 | 家长中心 | 学习报告/时间控制/使用监督 | P1 |
| 6 | 贴纸收集 | 贴纸册/稀有度/交换 | P2 |
| 7 | 排行榜 | 好友/班级/全国排行 | P2 |
| 8 | 挑战赛 | 好友对战/排位赛 | P2 |
| 9 | 离线缓存 | SQLite本地存储/同步策略 | P1 |

| 11 | 后台管理系统 | 内容管理/宠物/成就/装饰/审核 | P0 |

**开发周期估算**：每个P0/P1模块约1-2周，P2模块约1周。
